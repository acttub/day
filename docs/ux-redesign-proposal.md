# 오늘의 배우훈련 UX 리디자인 제안

## 0. 제품 원칙

**한 문장:** 오늘 10분 동안 바로 해볼 배우훈련 하나를 고르고, 따라 하고, 한 문장으로 남기는 모바일 웹 서비스.

**MVP 범위**
- 포함: 랜딩, 오늘 훈련 선택/추천, 결과 상세, 실행 가이드, 1분 기록, 최근 훈련, 공유/리텐션.
- 제외: 장면 분석, 영상 업로드/촬영, 연기 평가, 점수, 강점/약점, 코어 acttub CTA.
- 톤: 선생님/심사위원이 아니라 “연습을 같이 시작하게 해주는 작은 파트너”.

**핵심 UX 약속**
1. “뭐 하지?”를 10초 안에 해결한다.
2. 어려운 훈련 용어를 먼저 설명하지 않는다. 사용자가 할 행동부터 보여준다.
3. 결과 화면은 읽는 문서가 아니라 바로 따라 하는 타이머형 가이드에 가깝게 만든다.
4. 끝나고 남기는 기록은 부담 없는 한 문장이어야 한다.
5. 공유는 자랑보다 “오늘 이거 같이 해볼래?”에 가깝게 만든다.

## 1. 전체 정보구조

```txt
랜딩
  └─ 온보딩/선택
       ├─ 오늘 바로 추천받기
       ├─ 몸 깨우기
       ├─ 집중 정리하기
       ├─ 감각 열기
       ├─ 반응 연습하기
       └─ 상상 이어가기
            ↓
훈련 결과 상세
  ├─ 오늘의 훈련 요약
  ├─ 준비물/공간/혼자 가능 여부
  ├─ 10분 실행 가이드
  ├─ 하다가 막힐 때
  ├─ 끝나고 남길 질문
  └─ 시작/기록/공유 CTA
            ↓
실행 가이드 모드
  ├─ 0~2분
  ├─ 2~5분
  ├─ 5~8분
  └─ 8~10분
            ↓
1분 기록
  ├─ 실제로 해본 정도
  ├─ 오늘 남은 감각/발견
  ├─ 내일 다시 받을지
  └─ 저장 후 리텐션/공유
```

## 2. 랜딩 화면

### 현재 문제
- “오늘의 배우훈련!”은 명확하지만, 사용자가 얻는 결과가 조금 추상적이다.
- 카테고리명이 초보 배우에게는 먼저 어렵게 보일 수 있다.
- 첫 화면에서 “왜 매일 들어올 이유가 있는지”가 약하다.

### 리디자인 목표
- 첫 5초 안에 “오늘 10분짜리 연습 하나를 받는 서비스”라고 이해하게 한다.
- 전문 용어보다 상황형 문장으로 접근한다.
- CTA는 하나만 강하게 둔다.

### 화면 구조

```txt
[상단 작은 로고]
오늘의 배우훈련

[히어로]
오늘 연습, 뭐 할지 고민하지 마세요.
10분 동안 바로 따라 할 배우훈련 하나를 골라드릴게요.

[주요 CTA]
오늘 훈련 받기

[보조 선택]
오늘 몸 상태로 고르기
- 몸을 깨우고 싶어요
- 집중을 정리하고 싶어요
- 감각을 열고 싶어요
- 반응을 연습하고 싶어요
- 상상을 이어가고 싶어요

[신뢰/범위 안내]
대본 없어도 가능 · 대부분 혼자 가능 · 점수 없음 · 매일 다른 훈련

[최근 기록이 있는 경우]
지난번에는 “공간 속도 바꾸기”를 했어요.
오늘은 다른 감각으로 시작해볼까요?
```

### 권장 카피

**헤드라인**
- 기존: `오늘의 배우훈련!`
- 제안: `오늘 연습, 뭐 할지 고민하지 마세요.`

**서브카피**
- 기존: `오늘 10분 동안 뭐 할지 모르겠다면...`
- 제안: `대본이 없어도, 상대가 없어도 할 수 있는 10분 배우훈련을 하나 골라드릴게요.`

**CTA**
- Primary: `오늘 훈련 받기`
- Secondary: `몸 상태로 고르기`

**안내 뱃지**
- `10분이면 충분해요`
- `혼자서도 가능해요`
- `점수 없이 기록만 남겨요`
- `배우훈련만 나와요`

## 3. 온보딩/선택 화면

### 왜 필요한가
완전 랜덤은 빠르지만, 배우가 “오늘 몸이 무겁다 / 집중이 안 된다 / 상대 반응이 필요하다”처럼 현재 상태를 알고 있을 때는 선택권이 있어야 재방문 이유가 생긴다.

### MVP 방식
회원가입 없이 1문항 선택만 둔다. 선택하지 않으면 랜덤 추천.

### 화면 구조

```txt
[제목]
오늘은 어떤 연습이 필요하세요?

[설명]
잘 몰라도 괜찮아요. 가장 가까운 걸 고르면 됩니다.

[선택 카드]
1. 몸이 굳어 있어요
   → 걷기, 멈춤, 자세, 공간 훈련
   버튼: 몸 깨우기

2. 머리가 복잡해요
   → 숨, 시선, 첫 문장 준비
   버튼: 집중 정리하기

3. 감각이 닫힌 느낌이에요
   → 소리, 색, 사물, 온도 관찰
   버튼: 감각 열기

4. 혼자 말만 하는 느낌이에요
   → 듣기, 반복, 반응 훈련
   버튼: 반응 연습하기

5. 상상이 잘 안 이어져요
   → 즉흥, 사물, 상태 바꾸기
   버튼: 상상 이어가기

[하단]
잘 모르겠어요 → 랜덤으로 받을게요
```

### 용어 매핑
- `뷰포인트 영감`은 내부 카테고리로 유지하되 사용자 첫 선택에는 `몸/공간`으로 표시.
- `마이즈너 영감`은 첫 선택에서 `듣기/반응`으로 표시.
- `호흡/집중`은 `시작 전 정리`로 설명.
- 결과 상세에서만 `마이즈너 영감`처럼 작은 태그로 노출하고, “정식 수업을 대체하는 설명이 아니라 오늘의 연습 힌트”라고 부드럽게 처리.

## 4. 훈련 결과 상세 화면

### 현재 문제
현재 결과는 필요한 정보가 모두 있지만, 사용자는 어디서부터 시작해야 하는지보다 설명을 먼저 읽게 된다.

### 리디자인 목표
- 상단에서 “오늘 할 일”을 한눈에 보여준다.
- 준비물/공간/혼자 가능 여부를 먼저 알려 불안을 낮춘다.
- 10분 진행법은 카드형 단계로 분리한다.
- `훈련 해봤다`보다 `10분 시작하기`를 먼저 둔다.

### 화면 구조

```txt
[상단]
← 다른 훈련 보기
오늘 2026.07.06 · 몸/공간 훈련

[히어로 카드]
공간 속도 바꾸기
몸이 먼저 깨어나게 하는 10분 훈련

[빠른 정보]
소요 10분 · 준비물 없음 · 혼자 가능 · 방 안에서 가능

[Primary CTA]
10분 시작하기
[Secondary]
전체 진행법 읽기

[왜 이 훈련을 하나요]
걷는 속도와 멈춤을 바꾸면, 감정보다 먼저 몸의 리듬이 살아나는 순간을 찾을 수 있어요.

[10분 진행법]
0~2분: 아주 느리게 걸으며 시선이 가는 곳 3곳 찾기
2~5분: 빠르게 걷고 방향을 갑자기 3번 바꾸기
5~8분: 느림/빠름/멈춤을 섞어보기
8~10분: 몸이 가장 깨어난 속도로 마지막 30초 걷기

[하다가 막히면]
- 잘하려고 하지 말고 속도 차이만 크게 만들어보세요.
- 감정을 만들 필요 없어요. 몸이 먼저 반응하는지 보면 됩니다.
- 공간이 좁으면 제자리 방향 전환만 해도 됩니다.

[끝나고 남길 질문]
내 몸은 어떤 속도에서 가장 먼저 깨어났나요?

[하단 고정 CTA]
10분 시작하기 · 기록 남기기 · 공유
```

### 권장 카피 변경

| 현재 | 제안 |
|---|---|
| 오늘 이걸 해보는 이유 | 왜 이 훈련을 하나요 |
| 10분 진행법 | 10분 동안 이렇게 해보세요 |
| 할 때 기억할 것 | 하다가 막히면 이렇게 해보세요 |
| 끝나고 체크 | 끝나고 한 가지만 남겨보세요 |
| 훈련 해봤다 | 기록 남기기 |
| 결과 복사하기 | 진행법 복사 |
| 공유 문구 복사 | 친구에게 보내기 |

## 5. 실행 가이드 모드

### 왜 필요한가
결과 상세는 읽기 화면이고, 실제 연습 중에는 손에 들고 보기 쉬운 “한 단계씩 넘기는 화면”이 필요하다. 타이머 구현이 부담되면 MVP에서는 수동 단계 넘김으로 충분하다.

### 화면 구조

```txt
[상단]
공간 속도 바꾸기
1/4 · 0~2분

[큰 지시문]
방 안을 아주 느리게 걸어보세요.
시선이 자연스럽게 꽂히는 지점 세 곳을 찾습니다.

[작은 힌트]
예쁘게 걷는 게 아니라, 느림을 끝까지 느끼는 시간이에요.

[버튼]
다음 단계로

[하단]
중간에 멈춰도 괜찮아요 · 전체 진행법 보기
```

### 단계별 UX
- 1단계는 가장 쉬운 행동으로 시작한다.
- 각 단계는 최대 2문장만 보여준다.
- `다음 단계로` 버튼을 크게 둔다.
- 마지막 단계 완료 후 바로 `오늘 한 문장 남기기`로 연결한다.

### 실행 모드 카피 예시

**시작 전**
`휴대폰은 옆에 두고, 화면은 가끔만 봐도 괜찮아요.`

**진행 중**
`지금은 맞게 하는 시간이 아니라, 몸이 어떻게 반응하는지 보는 시간이에요.`

**완료**
`10분 끝. 지금 남아 있는 감각 하나만 적어볼까요?`

## 6. 1분 기록 / Post-practice Log

### 현재 문제
현재 기록 문항은 유용하지만 약간 설문처럼 느껴진다. 기록은 “서비스를 위한 데이터 입력”보다 “내 연습 노트”처럼 느껴져야 한다.

### 리디자인 목표
- 30~60초 안에 끝난다.
- 사용자가 못 했어도 이탈하지 않게 한다.
- 내일 다시 들어올 이유를 만든다.

### 화면 구조

```txt
[제목]
오늘 연습에서 남은 것

[설명]
길게 쓰지 않아도 괜찮아요. 한 단어만 남겨도 기록이 됩니다.

[문항 1: 실제 진행]
오늘 훈련을 어디까지 해봤나요?
- 10분 끝까지
- 5분 정도
- 조금만 맛봤어요
- 아직 안 했어요

[문항 2: 남은 감각]
끝나고 가장 남아 있는 건 무엇인가요?
- 몸
- 숨
- 시선
- 소리/감각
- 상상
- 잘 모르겠어요

[문항 3: 한 문장]
오늘 남은 한 문장
placeholder: 예: 빠르게 걷다가 멈출 때 몸이 제일 깨어났다.

[저장 CTA]
내 연습 노트에 저장

[저장 후]
저장됐어요.
내일은 오늘과 다른 훈련을 받을 수 있어요.
[내일 알림 문구 복사] [오늘 훈련 공유]
```

### 기록 저장 후 리텐션 카피
- `내일도 10분만 이어가볼까요?`
- `오늘은 몸/공간을 했어요. 내일은 감각이나 반응 훈련을 받아볼 수 있어요.`
- `알림은 아직 없지만, 이 문구를 캘린더나 메모장에 붙여둘 수 있어요.`

## 7. 리텐션 / 공유

### 리텐션 MVP
복잡한 계정/푸시 없이 로컬 저장과 복사 기능으로 시작한다.

**로컬 저장 화면**
```txt
최근 7일 연습
월 공간 속도 바꾸기
화 소리 5개 듣고 반응하기
수 낮은 소리로 첫 문장 5번

이번 주에 많이 연습한 감각: 몸/공간
다음에는 감각/관찰 훈련도 받아볼 수 있어요.
```

주의: `많이 연습한 감각`은 평가가 아니라 기록 요약이다. `잘함/부족함`으로 해석될 표현은 쓰지 않는다.

### 공유 UX
공유는 결과 전체보다 “같이 할 수 있는 짧은 초대장”이 좋다.

**버튼명**
- `친구에게 보내기`
- `오늘 훈련 복사`

**공유 문구 제안**
```txt
오늘의 배우훈련
「공간 속도 바꾸기」

10분 동안 걷는 속도와 멈춤을 바꿔보는 훈련.
오늘 질문: 내 몸은 어떤 속도에서 가장 먼저 깨어났나요?

나랑 오늘 이거 같이 해볼래?
```

**기록 후 공유 문구**
```txt
오늘 10분 배우훈련 완료.
남은 한 문장: “빠르게 걷다가 멈출 때 몸이 제일 깨어났다.”

내일도 하나만 해볼 예정.
```

## 8. 구현 우선순위

### P0 — 이번 MVP에서 바로 반영
1. 랜딩 헤드라인/서브카피 교체.
2. 결과 상세 상단에 `소요 시간 · 준비물 · 혼자 가능 · 공간` 빠른 정보 추가.
3. `10분 시작하기` CTA 추가.
4. `할 때 기억할 것`을 `하다가 막히면`으로 변경.
5. 기록 화면을 설문형에서 연습 노트형으로 카피 변경.
6. 공유 문구에서 `acttub` 단독 브랜드 노출보다 “오늘의 배우훈련” 초대장 중심으로 변경.

### P1 — 스타트업 MVP 느낌 강화
1. 온보딩 선택 화면 추가: 현재 상태 기반 5개 선택지.
2. 실행 가이드 모드 추가: 4단계 수동 진행.
3. 최근 7일 기록 요약 추가.
4. 기록 완료 후 `내일 다시 하기` 문구/캘린더 복사용 문구 제공.

### P2 — 검증 이후
1. 사용자가 선호하는 훈련 종류 기반 추천 가중치.
2. 훈련별 난이도/공간 제약 필터. 단, 난이도는 평가처럼 보이지 않게 `처음 하기 쉬움 / 몸을 조금 더 씀` 등 행동 기준으로 표현.
3. 계정/알림/주간 루틴. 단, 현재 MVP에서는 로컬 저장으로 충분.

---

## 9. 금지/주의 카피

사용하지 말 것:
- `평가`, `점수`, `실력`, `강점`, `약점`, `교정`, `처방`, `진단 결과`
- `당신의 연기를 분석합니다`
- `영상을 올려보세요`
- `장면 피드백 받기`
- `acttub에서 더 깊게 분석하기`

대신 쓸 것:
- `기록`, `오늘 남은 것`, `연습 노트`, `몸이 반응한 순간`, `다음에 가져갈 감각`
- `바로 해볼 훈련`, `10분 진행법`, `하다가 막히면`, `끝나고 한 문장`

---

## 10. 성공 지표

MVP에서 볼 지표:
1. 랜딩 → 훈련 결과 도달률
2. 결과 → `10분 시작하기` 클릭률
3. 결과 → 기록 저장률
4. 기록 중 `10분 끝까지/5분 정도` 비율
5. 공유 문구 복사율
6. 7일 내 재방문 수

해석 주의:
- 클릭률은 관심 지표이고, 실제 가치 지표는 `기록 저장`과 `재방문`이다.
- 평가/점수 없이도 사용자가 다시 오면 “오늘 하나 정해주는” 가치가 검증된다.

---

## 11. 최종 권장 방향

현재 구현은 “배우훈련만 추천한다”는 범위가 잘 지켜져 있다. 다음 리디자인은 기능을 많이 늘리기보다, 사용자가 모바일에서 바로 움직이게 만드는 구조로 바꾸는 것이 핵심이다.

가장 먼저 바꿀 것:
1. 랜딩을 더 명확한 문제/약속 중심으로 교체.
2. 결과 상세를 `요약 → 준비 정보 → 10분 시작 → 단계 가이드 → 한 문장 기록` 순서로 재배치.
3. 기록과 공유를 설문/복사 기능이 아니라 “내 연습이 남는 순간”으로 느끼게 만든다.
