리뷰 페르소나: 연기 입시생 / 초급 배우 / 바쁜 배우 지망생

## 1. 한 줄 결론

- **ready for 5-user test, but fix-first recommended for result CTA and record completion.**
- 핵심 루프(랜딩 → 상태 선택 → 훈련 결과 → 10분 실행 모드 → 30초 기록)는 실제로 완료 가능하다.
- 다만 결과 페이지가 아직 `읽는 문서`처럼 길고, 기록 완료 후 성취/재방문 장치가 약하다.

## 2. 실제 사용 흐름 결과

| Step | Result | Evidence |
|---|---|---|
| Landing | 통과 | 첫 화면 문구: `오늘 뭘 연습할지 고민하지 마세요.` / `어려운 배우훈련을 오늘 바로 따라 할 수 있는 10분 루틴으로 바꿔드립니다.` |
| State choice | 통과 | `집중 정리` 선택 가능. 선택지는 몸/집중/말/상상/반응/랜덤으로 쉬운 언어임. |
| Training result | 부분 통과 | `30초 정지 후 시작하기` 결과 표시. `이 훈련은 뭐예요?`, `언제 하면 좋아요?`, `10분 진행법`이 충분히 구체적임. 단, `10분 시작하기` CTA가 상세 설명 아래쪽에 있어 늦게 보임. |
| 10-minute mode | 통과 | `1/4 · 0~2분` → `4/4 · 8~10분` 단계 진행 가능. 마지막에 `기록 남기기` 표시됨. |
| 30-second record | 부분 통과 | 기록 저장은 localStorage에 완료됨. 저장 데이터: didPractice/bodyChanged/wantAgain/reflection/trainingId. 단, 저장 후 명확한 완료/축하/내일 재방문 화면이 약함. |
| Share/copy | 부분 통과 | `공유 문구 복사` 버튼은 존재하고 콘솔 에러는 없음. 단, 클릭 후 사용자에게 보이는 성공 피드백이 약하거나 없음. |

콘솔: 페이지 로드 및 주요 상호작용 후 JS 에러 없음.

## 3. Persona별 판단

### 연기 입시생

- **판단: 쓸 수 있음.**
- 수업 없는 날 `오늘 뭘 하지?`를 줄여주는 약속은 명확하다.
- `언제 하면 좋아요?`와 `할 때 기억할 것`이 있어서 혼자 해볼 근거가 생긴다.
- 다만 입시생은 빨리 시작하고 싶으므로 결과 상단에 `10분 시작하기`가 더 빨리 보여야 한다.

### 초급 배우 / 연기학원 수강생

- **판단: 이해 가능성이 높음.**
- `호흡/집중` 같은 카테고리는 보이지만, 결과 설명은 비교적 쉬운 말로 풀려 있다.
- `정지는 굳는 것이 아니라 듣는 시간입니다` 같은 문장은 배우훈련의 감각을 잘 번역한다.
- 다만 `정지 상태에서 충동이 생긴 뒤 시작하는 훈련입니다`는 초급자에게 약간 추상적이다. 더 행동형 문장으로 바꾸면 좋다.

### 바쁜 배우 지망생

- **판단: 첫 시작은 빠르지만 결과 페이지에서 이탈 가능.**
- 랜딩의 상단 CTA로 30초 안에 훈련을 받을 수 있다.
- 하지만 결과 페이지에서 실행 CTA가 아래쪽에 있어 바쁜 사용자는 읽다가 이탈할 수 있다.
- 실행 모드는 한 화면에 한 단계만 보여줘서 좋다.

## 5. 가장 큰 문제 3개

### 1. 결과 페이지가 아직 너무 `읽는 문서`처럼 시작된다

- Type: Conversion leak
- Evidence: 결과에서 `10분 시작하기`가 `이 훈련은 뭐예요?`, `언제 하면 좋아요?`, `이유`, `10분 진행법`, `잘하려고 하지 말고 볼 것`, `흔히 헷갈리는 점`, `할 때 기억할 것`, `안전하게 하기`, `다음 행동` 뒤에 나온다.
- Why it matters: 바쁜 사용자는 추천을 받은 뒤 바로 실행하고 싶다. 설명은 필요하지만 CTA가 늦으면 실행 전 이탈한다.
- Fix: hero card 바로 아래에 `10분 시작하기` primary CTA를 한 번 더 노출하고, 자세한 설명은 아래에 둔다.
- Acceptance check: 결과 화면 첫 viewport 안에 훈련명 + `10분 시작하기`가 같이 보인다.

### 2. 기록 저장 후 성취감/재방문 장치가 약하다

- Type: Conversion leak
- Evidence: localStorage 저장은 확인됐지만 사용자에게 `저장됐어요`, `내일 다시 오세요`, `최근 기록` 같은 명확한 완료 상태가 약하다.
- Why it matters: 이 제품의 activation은 `훈련 완료 + 기록`이다. 기록 후 보상이 없으면 다음날 돌아올 이유가 약해진다.
- Fix: 저장 후 별도 완료 화면 또는 완료 카드 추가: `오늘 훈련 기록 완료`, `내일 다시 하나 받을까요?`, `최근 기록 1개`.
- Acceptance check: 기록 저장 직후 완료 메시지와 내일 재방문 CTA가 보인다.

### 3. 공유/복사 후 피드백이 약하다

- Type: Polish / Growth leak
- Evidence: `공유 문구 복사` 클릭 후 콘솔 에러는 없지만 눈에 띄는 성공 상태가 보이지 않았다.
- Why it matters: 공유 루프는 초기 성장 가설의 핵심인데, 사용자가 복사 성공 여부를 모르면 공유 행동이 끊긴다.
- Fix: 버튼 클릭 후 `복사됐어요. 배우 친구에게 보내보세요.` 토스트 또는 버튼 상태 변경.
- Acceptance check: 공유 복사 후 2초 이상 보이는 성공 메시지가 표시된다.

## 6. 바로 고칠 UX/카피

- `정지 상태에서 충동이 생긴 뒤 시작하는 훈련입니다.`
  → `바로 움직이지 않고 30초 멈춘 뒤, 몸에서 먼저 생기는 작은 시작 신호를 기다려보는 훈련입니다.`

- 결과 페이지 CTA 위치
  → hero card 아래에 `10분 시작하기` 추가, 하단 CTA는 유지

- 기록 저장 후
  → `기록 저장됨. 내일은 다른 훈련으로 이어갈 수 있어요.`

- 공유 성공
  → `복사됐어요. 배우 친구에게 오늘 훈련을 보내보세요.`

## 7. 다음 실험

5명에게 링크를 보내고 아래만 측정한다.

1. 첫 방문에서 `오늘 훈련 바로 받기`를 눌렀는가?
2. 결과를 보고 `10분 시작하기`까지 갔는가?
3. 4/4 단계까지 갔는가?
4. 30초 기록을 남겼는가?
5. 끝나고 `내일도 받을 의향`에 예/상황에 따라를 골랐는가?

성공 기준:

```txt
5명 중 4명: 오늘 훈련 받기
5명 중 3명: 10분 시작하기
5명 중 2명: 30초 기록 작성
5명 중 2명: 내일도 받을 의향 있음
```
